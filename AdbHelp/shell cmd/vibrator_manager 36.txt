Vibrator Manager commands:
  help
    Prints this help text.
  list
    Prints device vibrator ids; does not include input devices.
  synced [options] <effect>...
    Vibrates effect on all vibrators in sync.
  combined [options] (-v <vibrator-id> <effect>...)...
    Vibrates different effects on each vibrator in sync.
  xml [options] <xml>
    Vibrates using combined vibration described in given XML string
    on all vibrators in sync. The XML could be:
        A single <vibration-effect>, or
        A <vibration-select> containing multiple effects.
  feedback [options] <constant>
    Performs a haptic feedback with the given constant.
  cancel
    Cancels any active vibration

Effect commands:
  oneshot [-a] <duration> [<amplitude>]
    Vibrates for duration milliseconds.
    If -a is provided, the command accepts a second argument for
    amplitude, in a scale of 1-255.
  waveform [-r index] [-a] [-f] [-c] (<duration> [<amplitude>] [<frequency>])...
    Vibrates for durations and amplitudes in list.
    If -r is provided, the waveform loops back to the specified
    index (e.g. 0 loops from the beginning).
    If -a is provided, the command expects amplitude to follow each
    duration; otherwise, it accepts durations only and alternates
    off/on.
    If -f is provided, the command expects frequency to follow each
    amplitude or duration; otherwise, it uses resonant frequency.
    If -c is provided, the waveform is continuous and will ramp
    between values; otherwise each entry is a fixed step.
    Duration is in milliseconds; amplitude is a scale of 1-255;
    frequency is an absolute value in hertz.
  envelope [-a] [-i initial sharpness] [-r index]  [<duration1> <intensity1> <sharpness1>]...
    Generates a vibration pattern based on a series of duration,
    intensity, and sharpness values. The total vibration time is
    the sum of all durations.
    If -a is provided, the waveform will use the advanced APIs to
    generate the vibration pattern and the input parameters
    become [<duration1> <amplitude1> <frequency1>].
    If -i is provided, the waveform will have an initial sharpness
    it will start from.
    If -r is provided, the waveform loops back to the specified index
    (e.g. 0 loops from the beginning).
  prebaked [-b] <effect-id>
    Vibrates with prebaked effect.
    If -b is provided, the prebaked fallback effect will be played if
    the device doesn't support the given effect-id.
  primitives ([-w delay] [-o time] [-s scale]<primitive-id> [<scale>])...
    Vibrates with a composed effect.
    If -w is provided, the next primitive will be played after the
    specified wait time in milliseconds.
    If -o is provided, the next primitive will be played at the
    specified relative start offset time in milliseconds.
    If -s is provided, the next primitive will be played with the
    specified amplitude scale, in a scale of [0,1].

Common Options:
  -f
    Force. Ignore Do Not Disturb setting.
  -B
    Run in the background; without this option the shell cmd will
    block until the vibration has completed.
  -S
    Run vibration in a vendor session. Only vibration commands will
    apply this option. The -B option will be ignored.
  -u <usage>
    Specify the usage for the haptic feedback or vibration.
  -d <description>
    Add description to the vibration.

Notes
    Vibrations triggered by these commands will be ignored when
    device is on DND (Do Not Disturb) mode; notification strength
    user settings will be applied for scale.
